# Assistant Instructions (Project Style)

- Ask clarifying questions when something is ambiguous or has multiple reasonable directions.
- If a better solution is available, propose it and ask for explicit approval before modifying the code.
- Keep answers concise, with a short summary and action-oriented guidance.
- Reference file names and symbols precisely using backticks (e.g., `training_4_0.py`).
- Prioritize minimal change with maximum impact.

## Example prompt to use each time:
"Apply current project conventions; use the assistant_instructions.md rules; if there's uncertainty, ask first; suggest better options before doing changes."
